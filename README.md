> **工业级、全栈式、堡垒化远程运维管理平台**

天穹 是一款专为运维工程师打造的、“端云一体”的移动级 Linux 堡垒机。它集成了高安全性的 SSH 隧道、流式文件管理、实时监控、文件传输记录监控及 Android 原生隐私保护能力。

---

## 1. 产品核心定位与功能 (Product Features)

### 1.1 极简接入与安全加固 (Security & Access)
- **密钥驱动自适应授权**：App 启动后仅需输入访问密钥 (Admin Key)。零配置界面，实现秒级快速登录，同时对外隐藏敏感连接信息。
- **物理隐身模式 (Stealth Mode)**：独创 H5 管理面板开关。当在 App 中关闭开关后，外部任何人访问服务端 Web 端口均会直接返回逼真的 HTTP 404 Not Found 页面，完美伪装，躲避扫描探测。
- **物理数据擦除 (Wipe Data)**：在 Android 客户端顶部设有红色**紧急退出按钮**。点击后将不仅清空本地设备上的所有凭据痕迹，还会向服务器发送销毁指令，擦除服务器上所有缓存的书签、配置与敏感日志，实现物理隔离断代。

### 1.2 高性能终端与监控 (Terminal & Monitor)
- **全彩 ANSI 终端 (SSH)**：完整支持 ANSI 转义序列、16/256色，原生兼容 `top`, `htop`, `vim`。并提供移动端专用的物理控制悬浮键 (ESC, Tab, Ctrl 等)。
- **智能重置 (Terminal Reset)**：终端下方内置一键“重置连接”按钮。在发生网络波动、阻塞时，可一键清空缓存并重启 WebSocket 隧道。
- **全景监控看板**：首页实时展示服务器 Uptime、延迟 RTT、CPU负载、内存占用及各个挂载磁盘空间的动态利用率。
- **书签库直连**：提供独立配置面板，可将高频主机存入书签，一键参数回填并自动连接。

### 1.3 文件流转与传输追踪 (SFTP & Transfer)
- **可视化 SFTP 引擎**：支持全盘目录漫游浏览，多层级快速穿透。
- **双端传输监控中心**：全新引入的独立 “传输 (Transfer)” 监控页。通过队列管理所有上传/下载任务，实时查看传输状态、字节进度与失败错误（Android/H5 数据完全云端持久化互通）。
- **路径协同互通**：在文件浏览器中点击“同步终端”，对应的 SSH 终端即可自动 `cd` 切换到当前目录。

### 1.4 自动化与全息审计 (Audit & Automation)
- **命令片段库 (Snippets)**：预设运维高频指令，通过卡片式标签页点击即刻注入终端。
- **全量行为审计录像**：详尽记录所有登录请求、文件变动、连接断开等行为。数据可在多端同步溯源。

## 2. 部署与安装指南 (Deployment & Installation)

项目的所有最终发行版均位于代码根目录下的 `release_v1.0_stable/` 文件夹中。为方便快速上线，已提供了完整的 **`安装包.zip`**。

### 2.1 推荐部署方式 (Docker Compose 容器化)
天穹项目深度支持无环境依赖的 Docker 容器化部署，这不仅简化了运行环境，还能通过文件映射**极大地简化外部配置的二次修改**。

**第 1 步：准备部署文件**
请确保宿主机已安装 Docker 及 Docker Compose。将发行版 `release_v1.0_stable/` 中的以下三个文件提取至宿主机的同一个空目录下：
- `sty-visit.jar` (核心业务服务)
- `application.yml` (外部配置基础模板)
- `docker-compose.yml` (容器挂载配置)

**第 2 步：如何引入并修改外部配置文件？**
在 `docker-compose.yml` 中，已经通过 `volumes` 参数自动做好了文件绑定的映射关系（`- ./application.yml:/app/application.yml`）。
因此，您**不需要把配置打进镜像中**，只需按如下步骤操作：
1. **直接修改外部的 `application.yml`**：使用任何文本编辑器打开宿主机上的 `application.yml`，修改诸如管理员密码（`styvisit.admin-password`）等信息并保存。
2. **免打包直接生效**：容器在下次启动时，会自动读取您外部这套修改后的最新配置，从而实现外置注入。

**第 3 步：一键后台启动**
在放有上述文件的目录内，执行：
```bash
docker-compose up -d
```

**第 4 步：状态与持久化说明**
启动后，您会在当前目录下看到自动生成的 `config/` 和 `logs/` 文件夹。这意味着你的业务状态、书签以及日志已经被**自动持久化**隔离出来，即使后期你通过 `docker-compose down` 删除容器并更新了镜像，只要这些目录和外置的 `application.yml` 还在，配置和数据就永远不会丢失。

### 2.2 传统服务端部署 (Java Backend)
1. **解压安装包**：在您的 Linux/Unix 服务器上解压 `安装包.zip`。
2. **环境要求**：确保服务器已安装 JRE 8 或 JDK 1.8 及以上版本。
3. **配置文件**：通过修改目录下的 `application.yml` 调整：
   - `server.port`: 服务监听端口 (默认 8080)
   - `styvisit.admin-password`: 您的全局超级密钥 (必须牢记！)
4. **一键启动**：
   ```bash
   chmod +x run.sh
   ./run.sh
   ```
   *服务将在后台静默运行，并产生 `logs/sty-visit.log` 运行日志。默认开启 8080 端口。*

### 2.3 客户端安装 (Android App)
1. 从 `release_v1.0_stable` 或解压后的压缩包内提取 `sty-visit.apk`。
2. 发送至您的安卓手机，允许安装未知来源应用，点击安装。

## 3. 操作与使用向导 (User Guide)

### 3.1 初始登录与隐身控制
1. **登录系统**：打开 App，在服务器地址栏输入 `http://您的服务器IP:8080`，并输入在 `application.yml` 中设置的密码，点击登录。
2. **Web 端口隐身**：进入 **"监控"** 页面。在此页面您可以找到 **"H5 管理面板开关"**。
   - **开启**：此时可以通过电脑浏览器访问 `http://您的服务器IP:8080` 进行网页管理。
   - **关闭**：强制阻断 Web 端。任何浏览器访问均会触发 404 Not Found 的白板伪装页面，实现端口级深度隐身。

### 3.2 终端操作与紧急擦除
1. **书签连接**：进入“终端”页，可以在上方直接填入主机密码，也可点击下方“书签”按钮一键调出配置好的服务器并连接。
2. **特殊控制键**：连接成功后，屏幕键盘上方会出现横向滑动的悬浮快捷栏。
3. **紧急退出擦除**：无论您处在 App 哪个界面，只要遇到紧急情况，直接点击**顶部导航栏右上角的红色退出图标**。此时您的客户端会立刻断开连接、清除本地内存，并销毁服务器上的会话凭证文件。

### 3.3 文件的上传与下载
1. 前往 **"文件"** 模块，系统会自动列出当前用户的根目录文件。
2. 长按任意文件或点击更多，选择“下载”。
3. 点击底部导航栏的 **"传输"**，可以实时查看所有排队、进行中、或已完成的任务日志。文件下载完成后将保存在手机自带的 `Downloads` 目录下。

## 4. 版本更新与平滑升级指南 (Upgrade Guide)

随着业务的迭代，未来如果您获取到了新版本的天穹发布包（如新版 `sty-visit.jar`），您可以按照以下方式进行无损升级，您的业务配置和历史书签数据**绝对不会丢失**：

### 4.1 Docker 容器化平滑升级
1. **停止老容器**：在包含 `docker-compose.yml` 的部署目录下，执行 `docker-compose down` 以销毁当前运行的容器。
2. **替换内核包**：将最新下载的 `sty-visit.jar` 覆盖替换掉目录中原本的旧 jar 包。
3. **重建镜像并拉起**：由于外部配置文件（`application.yml`）和业务状态（`config/` 目录）均受 Docker Volume 挂载保护，您仅需执行 `docker-compose up -d --build`。系统会基于新的内核重新构建镜像，并完美继承您此前的所有数据。

### 4.2 传统 Jar 直装版升级
1. **结束旧进程**：可通过 `ps -ef | grep sty-visit` 找到老的后台守护进程并结束它。
2. **替换内核包**：使用新版的 `sty-visit.jar` 替换目录中的老文件。
3. **重新启动**：再次执行 `./run.sh`。新版程序启动时会自动去读取同级目录下不变的 `application.yml` 与 `config/` 文件夹。
