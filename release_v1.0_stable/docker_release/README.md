# 天穹 (sty-visit v1.0) - Docker 容器版专属部署指南

欢迎使用天穹堡垒机。本包是为您准备的 **Docker 容器化发出版**，推荐所有具备 Docker 运行环境的用户使用，它能最大程度地将运行环境与您的宿主机隔离开来。

## 📦 包含的文件说明
- `docker-compose.yml`：核心的容器编排文件，负责资源挂载和一键启动。
- `Dockerfile`：用于在本地将 JAR 包打为无 JDK 的纯粹 JRE 镜像。
- `sty-visit.jar`：天穹核心的编译产物。
- `application.yml`：外置的动态基础配置文件，您只需修改这个文件即可。

---

## 🛠️ 1. 如何修改配置文件？
容器化最大的优势是**配置与代码分离**。您不需要进入容器去修改任何东西！

1. 直接在宿主机使用文本编辑器打开同级目录下的 `application.yml`。
2. 找到并修改您的专属超级密钥（默认是 `StyVisit@2026`），这是所有访问的最高防线：
   ```yaml
   styvisit:
     admin-password: "您的复杂密码"
   ```
3. 您可以按需修改默认的 8080 端口映射。
4. **修改生效说明**：任何时候您修改了外部的 `application.yml`，只需执行 `docker-compose restart` 即可热重载配置。

---

## 🚀 2. 如何一键部署运行？

只需一行命令即可完成构建并挂载所有数据通道：

```bash
docker-compose up -d --build
```

**运行后的状态说明**：
- **数据不丢**：成功启动后，当前目录会自动生成 `config/`（存放动态凭据、书签和策略）与 `logs/`（系统运行日志）。这归功于 `docker-compose.yml` 中的 Volume 映射。即使您将来用 `docker-compose down` 销毁了容器，只要这几个文件/文件夹还在，您的配置永远存在！
- 访问：您可以直接通过浏览器或 App 访问 `http://您的服务器IP:8080` 验证是否成功启动。
